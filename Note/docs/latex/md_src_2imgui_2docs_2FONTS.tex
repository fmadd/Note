The code in \doxylink{imgui_8cpp}{imgui.\+cpp} embeds a copy of \textquotesingle{}Proggy\+Clean.\+ttf\textquotesingle{} (by Tristan Grimmer), a 13 pixels high, pixel-\/perfect font used by default. We embed it in the source code so you can use Dear \doxylink{namespaceImGui}{Im\+Gui} without any file system access. Proggy\+Clean does not scale smoothly, therefore it is recommended that you load your own file when using Dear \doxylink{namespaceImGui}{Im\+Gui} in an application aiming to look nice and wanting to support multiple resolutions.

You may also load external .TTF/.OTF files. In the \href{https://github.com/ocornut/imgui/tree/master/misc/fonts}{\texttt{ misc/fonts/}} folder you can find a few suggested fonts, provided as a convenience.

{\bfseries{A vast majority of font and text related issues encountered comes from 4 things\+:}}\hypertarget{md_src_2imgui_2docs_2FONTS_autotoc_md141}{}\doxysubsubsection{\texorpdfstring{(1) Invalid filename due to use of $<$tt$>$\textbackslash{}$<$/tt$>$ or unexpected working directory.}{(1) Invalid filename due to use of <tt>\textbackslash{}</tt> or unexpected working directory.}}\label{md_src_2imgui_2docs_2FONTS_autotoc_md141}
See \doxylink{imgui__impl__opengl3__loader_8h_a1e3b09934060e9d373139c3557730d37}{About Filenames}. Add\+Font\+XXX functions should assert if the filename is incorrect.\hypertarget{md_src_2imgui_2docs_2FONTS_autotoc_md142}{}\doxysubsubsection{\texorpdfstring{(2) Invalid UTF-\/8 encoding of your non-\/\+ASCII strings.}{(2) Invalid UTF-8 encoding of your non-ASCII strings.}}\label{md_src_2imgui_2docs_2FONTS_autotoc_md142}
See \doxylink{imgui__impl__opengl3__loader_8h_a1e3b09934060e9d373139c3557730d37}{About UTF-\/8 Encoding}. Use the encoding viewer to confirm encoding of string literal in your source code is correct.\hypertarget{md_src_2imgui_2docs_2FONTS_autotoc_md143}{}\doxysubsubsection{\texorpdfstring{(3) Missing glyph ranges.}{(3) Missing glyph ranges.}}\label{md_src_2imgui_2docs_2FONTS_autotoc_md143}
You need to load a font with explicit glyph ranges if you want to use non-\/\+ASCII characters. See \doxylink{imgui__impl__opengl3__loader_8h_a1e3b09934060e9d373139c3557730d37}{Fonts Loading Instructions}. Use \doxylink{imgui__impl__opengl3__loader_8h_a1e3b09934060e9d373139c3557730d37}{Debug Tools} confirm loaded fonts and loaded glyph ranges.

This is a current constraint of Dear \doxylink{namespaceImGui}{Im\+Gui} (which we will lift in the future)\+: when loading a font you need to specify which characters glyphs to load. All loaded fonts glyphs are rendered into a single texture atlas ahead of time. Calling either of {\ttfamily io.\+Fonts-\/\texorpdfstring{$>$}{>}Get\+Tex\+Data\+As\+Alpha8()}, {\ttfamily io.\+Fonts-\/\texorpdfstring{$>$}{>}Get\+Tex\+Data\+As\+RGBA32()} or {\ttfamily io.\+Fonts-\/\texorpdfstring{$>$}{>}Build()} will build the atlas. This is generally called by the Renderer backend, e.\+g. {\ttfamily \doxylink{imgui__impl__dx11_8cpp_ac732cf6c340b83e31fefa09cf396ef91}{Im\+Gui\+\_\+\+Impl\+DX11\+\_\+\+New\+Frame()}} calls it. {\bfseries{If you use custom glyphs ranges, make sure the array is persistent}} and available during the calls to {\ttfamily Get\+Tex\+Data\+As\+Alpha8()/\+Get\+Tex\+Data\+As\+RGBA32()/\+Build()}.\hypertarget{md_src_2imgui_2docs_2FONTS_autotoc_md144}{}\doxysubsubsection{\texorpdfstring{(4) Font atlas texture fails to upload to GPU.}{(4) Font atlas texture fails to upload to GPU.}}\label{md_src_2imgui_2docs_2FONTS_autotoc_md144}
This is often of byproduct of point 3. If you have large number of glyphs or multiple fonts, the texture may become too big for your graphics API. {\bfseries{The typical result of failing to upload a texture is if every glyph or everything appears as empty black or white rectangle.}} Mind the fact that some graphics drivers have texture size limitation. If you are building a PC application, mind the fact that your users may use hardware with lower limitations than yours.

Some solutions\+:
\begin{DoxyItemize}
\item You may reduce oversampling, e.\+g. {\ttfamily font\+\_\+config.\+OversampleH = 1}, this will half your texture size for a quality loss. Note that while OversampleH = 2 looks visibly very close to 3 in most situations, with OversampleH = 1 the quality drop will be noticeable. Read about oversampling \href{https://github.com/nothings/stb/blob/master/tests/oversample}{\texttt{ here}}.
\item Reduce glyphs ranges by calculating them from source localization data. You can use the {\ttfamily \doxylink{structImFontGlyphRangesBuilder}{Im\+Font\+Glyph\+Ranges\+Builder}} for this purpose and rebuilding your atlas between frames when new characters are needed. This will be the biggest win!
\item Set {\ttfamily io.\+Fonts.\+Flags \texorpdfstring{$\vert$}{|}= Im\+Font\+Atlas\+Flags\+\_\+\+No\+Power\+Of\+Two\+Height;} to disable rounding the texture height to the next power of two.
\item Set {\ttfamily io.\+Fonts.\+Tex\+Desired\+Width} to specify a texture width to reduce maximum texture height (see comment in {\ttfamily \doxylink{structImFontAtlas_a81e39e30dffa4dd7e458a53297451e27}{Im\+Font\+Atlas\+::\+Build()}} function).
\end{DoxyItemize}

\item Read documentation in the \href{https://github.com/ocornut/imgui/tree/master/misc/freetype}{\texttt{ misc/freetype/}} folder.

{\bfseries{Please note that many new C/\+C++ users have issues loading their files {\itshape because the filename they provide is wrong} due to incorrect assumption of what is the current directory.}}

Two things to watch for\+:

(1) In C/\+C++ and most programming languages if you want to use a backslash {\ttfamily \textbackslash{}} within a string literal, you need to write it double backslash {\ttfamily \textbackslash{}\textbackslash{}}. At it happens, Windows uses backslashes as a path separator, so be mindful.

\end{DoxyCode}
 In some situations, you may also use {\ttfamily /} path separator under Windows.

(2) Make sure your IDE/debugger settings starts your executable from the right working (current) directory. In Visual Studio you can change your working directory in project {\ttfamily Properties \texorpdfstring{$>$}{>} General \texorpdfstring{$>$}{>} Debugging \texorpdfstring{$>$}{>} Working Directory}. People assume that their execution will start from the root folder of the project, where by default it often starts from the folder where object or executable files are stored.
